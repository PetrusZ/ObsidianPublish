    # 笔记风格

    • 冰与火之歌作者乔治马丁，关于写作说过一段很形象的比喻
      • 存在两种作家，一种是建筑师，一种是园丁
      • 建筑师提前计划一切，园丁则负责埋下幻想的种子，然后任由其生根发芽
      • 他认为自己更像是园丁
    • 而 Youtube 作者 Tiago Frote 在建筑师和园丁的基础上又增加了两个分类
      • 图书管理员和学生
    • 建筑师逻辑清晰、目的明确，他们先构建地基，自下而上搭好框架，目标是能建造出一栋结构完整的笔记大厦
      • 比较传统的做法，传统的文件夹结构，结构体系明确
      • 但一开始搭建结构和持续整理的过程非常麻烦
    • 而园丁的思维则更加跳跃，更注重在笔记之间进行联想和探索，他们不知道发芽的种子会长出多少分枝，但在脉络交错的花园中，总能修剪出不期而遇的灵感美景
      • 新式的双向链接做法
    • 图书管理员的首要目标，则是搜集、收藏、然后管理，而管理的目的是为了能够便于后期的检索以及研究
    • 最终是学生，他们的记录更加功利，周期相对更短，无需、也没有时间涉及太过高深的笔记系统，简短、快速、便捷的记录方式最适合他们

    # Notion

    B 站最强 Notion 测评，极致丝滑！新手进阶必看 16 个教程用法丨笔记软件推荐

    # 优点

    1. 数据库、视图功能强大。
    2. 排版美观，非常漂亮.
    3. 使用便捷。
    4. 区块（block）
      1. 最小笔记单位
      2. 可拖动
      3. 可转换
      4. 可评论
      5. 可链接

    # 缺点

    1. 格式不通用，虽然能导出markdown，但会损失不少信息。
    2. 双向链接功能较弱
    3. 没有大纲。
    4. 有时会卡、慢。
    5. 不支持嵌入B站视频
    6. 支持的标题层级太少

    # 适合

    1. notion 适合系统的、线性的、有逻辑顺序的创作，例如日记、工作文档、学习笔记、搭建知识系统。

    2. notion 也适合用来组织、整理和分享收集和创建的素材和图文作品。

    3. 但 notion 不适合碎片化的记录方式。写作压力 会比较大，需要思考文件叫什么名字，放到哪里等。启动速度较慢也不适合记录转瞬即逝的灵感。

    # Obisidian

    也许是 B 站最好的 Obsidian 新手教程！爆肝30天，一站式入门双向链接笔记软件

    # 优点

    1. 双向链接功能强大
      1. 比 notion 强太多
      2. 关系图谱
    2. 支持传统的文件夹结构
    3. 插件丰富
    4. 存储格式 markdown，与原生 markdown 兼容性最好
    5. 完全离线
    6. 第三方同步、备份还有发布方案完善

    # 缺点

    1. 原生不支持分栏，插件实现分栏太麻烦。排版不方便、不美观。不支持数据库-视图？
    2. 部分插件处于没有人维护的状态，功能无法正常使用，或功能不好用
      1. 无法嵌入播放视频，只能在阅读模式中打开，并且插件还有bug，切换阅读和编辑模式会不断弹出新窗口
      2. PDF 读书笔记几乎无法使用，markMind 与 annotator 在编辑模式下显示元数据太多，可读性不好，并且 annotator 打开大 PDF 时非常卡，并且不方便实现跳转。
    3. 不支持拖拽区块，notion 和 logseq 都支持
    4. 插件配置复杂
      1. 建议先把笔记记下来，插件、方法论可以后面再配置、整理

    # 用法

    1. 在日记中记录一切事情
      1. 今天做了什么、待办事项、任何想记下的东西
      2. 不受限制也不用分类的记录在日志中，减轻写作压力
      3. 积少成多，零散的将内容记录在每天日记中，再通过双链将每天日记中值得关注点汇总
      4. daily note + 子弹笔记，两种最基础的记录笔记方法
    2. 用双链汇总关键信息
      1. 中括号框住关键词
      2. 点击为关键词创建文档
      3. 然后在文档的反向链接面板中，笔记库内的任意地方，提到过这个关键词的信息，都会出现在这里
      4. 轻松汇总关键信息
      5. 关系图谱
    3. 任务管理系统
      1. task 插件实现
      2. 在任意地方记录下待办事项，想记在哪就记在哪，没有压力
      3. 在特定地方实现汇总管理
    4. kanban 插件
      1. 更直观的管理项目工程，trello 简化版
    5. 读书笔记
      1. 有深度阅读和 PDF 批注需求，更高效地做读书笔记
      2. 串联笔记、用双链拓展读书笔记的可能性
    6. 视频笔记
      1. 支持 youtube、bilibli
      2. 支持时间戳跳转
    7. 思维导图
      1. 使用mindmap 插件
      2. 按照标题层级，自动生成思维导图
    8. 阅读清单整理
      1. dataview 插件
      2. 类型 SQL 查询方式
      3. 根据文档自动生成清单
    9. 记录日报周报

    # 插件

    # Logseq

    爆肝100小时，全网最易懂的 Logseq 指南！双向链接笔记软件科普

    # 优点

    1. block 为最小单位。非常方便。
    2. 大纲结构
    3. 超级日记本：支持嵌入代码、表格、视频、画板
    4. 有插件系统
      1. 支持flomo的热力图
      2. 支持将大纲转换成思维导图
    5. 开源
    6. 读书笔记和视频笔记功能做的比 obsidan 好
      1. PDF 读书笔记做的比 obsidian 好
      2. 可以嵌入视频做视频笔记，youtube、bilbili 支持时间戳跳转
        1. 嵌入的bilbili视频不支持1080
    7. 完全离线
    8. 有潜力

    # 缺点

    1. 插件太少
    2. 强制大纲模式，所有行开头都有 -
    3. 长文体验不好
    4. 没有文件夹结构
    5. 比较卡
    6. 多平台同步没有免费方案
    7. 原生不支持在侧边栏显示大纲（跟 notion 一样）

    # 用法

    1. 日记随便，跟 obsidian 类似
    2. 任务管理系统，跟 obsidian 类似
    3. 记录工作日报周报，跟 obsidian 类似
    4. 用双向链接串联碎片灵感，跟 obsidian 类似
    5. 读书笔记，官方支持，比 obsidian 做的好
    6. 视频笔记，官方支持 youtube，B 站需要插件
